# Jenkins build-zigbee2mqtt pipline

Pipline описанный в файле [Jenkinsfile](Jenkinsfile) собирает пакет\
`zigbee2mqtt` для архитектуры указанной в поле `WBDEV_TARGET`.

Если в поле TAG ничего не указать, то собирается zigbee2mqtt последнего\
тега который был выложен в репозиторий [Koenkk/zigbee2mqtt](https://github.com/Koenkk/zigbee2mqtt).\
Если нужно собрать специфичный тег, то нужно ввести в поле `TAG` название\
тега точно так же как на github, например `2.3.0`.

Для скачивания зависимостей используется testing репозиторий, данное\
поведение можно поменять убрав галочку из чекбокса `USE_TESTING_REPOSITORY`.

Из этого репозитория выкачивается nodejs версии указанной в поле\
`FPM_DEPENDS` и собирает с ее помошью пакет с через pnpm. После сборки\
удаляются зависимости нужные только для сборки и из полученного списка\
файлов собирается уже конечный deb пакет, который будет тестироваться\
или выкладываться в репозиторий.

При обновлении версий в репозитории WB нужно всегда собирать пакеты парами:

- Один для архитектуры *-armhf (32бит)
- Второй для архитектуры *-arm64 (64бит)

## VERSION_TO_NAME

Имя собираемого пакета будет указано как zigbee2mqtt, но при фиксации\
версии можно отметить чекбокс `VERSION_TO_NAME` - в этом случае для\
получившегося пакета будет сгенерировано имя с постфиксом\
вида `zigbee2mqtt-1.18.1`.

## UPLOAD_TO_POOL

Первый раз собираемый пакет нужно всегда тестировать и только после теста\
можно заливать его в наш testing репозиторий обновляя таким образом\
версии пакетов. Для безопасности изначально пакеты не заливаются\
в репозиторий. После тестирования нужно снова собрать два пакета (32/64 бит),\
но при этом отметить чекбокс `UPLOAD_TO_POOL`.

## WBDEV_IMAGE

В процессе разработки иногда выявляется, что не хватает функционала в wbdev.\
В этом случае правки вносятся сначала в тестинг образ wbdev и если они\
проверяются, то уже потом переносятся в основной образ. Поэтому бывает\
полезно изменять используемый образ wbdev на тестинг:

В поле WBDEV_IMAGE можно писать:
- `contactless/devenv:latest` - образ wbdev по дефолту, релизный
- `contactless/devenv_test:latest` - образ wbdev для тестирования

## Изменение версии nodejs

Если необходимо поднять версию nodejs, то ее нужно поднимать в двух местах:

- На сервере Jenkins, которая используется при сборке
- В зависимостях генерируемого пакета (файл control)

Для того чтобы сделать это необходимо изменить строку в выпадающем\
списке `FPM_DEPENDS`.
